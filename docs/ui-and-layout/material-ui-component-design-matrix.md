# Material UI Component Design Matrix

## Purpose

本文件是第一轮 Material UI 全组件样式设计表。它以本地 `@mui/material` v9.0.1 源码和 MUI `docs/data/material/components` 公开目录为来源，把 MUI 组件族翻译成 Zircon Editor retained UI 能表达的 dark Material 设计约定。

当前目标是冻结设计和验证口径，并把每个组件映射到一个 `material_<component>.zui` 单组件原型；不复制 React API。后续实现批次必须从这张表选择组件族，补齐主题 token、painter/command 表达、showcase 示例和视觉 + 交互验证。

2026-05-17 inventory freeze: the authoritative public source page is MUI v9.0.1 all-components (<https://mui.com/material-ui/all-components/>). Keep the existing 63 Core/Utils/Lab rows and the 11 listed MUI X extension rows, including Date and Time Pickers.

2026-05-23 MUI Web parity freeze: `dev/material-ui/package.json` is the local authority for the new ZirconEditor parity track. The inventory gate now pins `mui-material/src/index.js`, `mui-lab/src/index.js`, `mui-icons-material/lib`, and the retained MUI X prototype list before renderer or component behavior changes begin. This supersedes Slint Material as the visual/API authority for future ZirconEditor MUI work while keeping the editor no-direct-Slint fence intact.

The runtime catalog gate intentionally separates direct descriptors from grouped subcomponents. The M3 form/input subcomponent slice has promoted `ButtonBase`, `InputBase`, `FilledInput`, `OutlinedInput`, the FormControl family, `InputAdornment`, `InputLabel`, `NativeSelect`, `RadioGroup`, and `ScopedCssBaseline` to standalone descriptors because they are the structural basis for TextField, Select, Radio, Checkbox, and Inspector forms. The M3 collection and surface/navigation child slices then promoted the local MUI Web List/ImageList/Table children plus Accordion/Dialog/MenuList/MobileStepper/SpeedDial/SwipeableDrawer/Tabs scroll children to direct descriptor coverage. Remaining grouped rows are now Lab-only context/tree/timeline helpers and picker variants under `DateTimePickers`; each promotion must update the classification test in the same change.

M1 customization contract: every Material foundation descriptor now exposes `mui_variant`, `mui_color`, `mui_size`, `mui_slots`, `mui_slot_props`, `mui_sx`, and `mui_classes`. These fields mirror MUI Web's variant/color/size plus slot, slot-prop, `sx`, and class customization surface without importing React or Slint APIs. Component-specific descriptors may still add stricter typed props such as `button_variant` or `variant`; the `mui_*` fields are the common escape hatch and authoring bridge.

M1 style bridge: runtime template style application now projects the common MUI customization props into selector-visible metadata. `mui_sx` merges into node attributes and `style_overrides` after stylesheet rules, so it behaves like a high-priority inline style while preserving authored inline overrides above it. The same pass generates `Mui<Component>-root`, variant/color/size utility classes, custom `mui_classes`, and MUI state classes such as `Mui-disabled`, `Mui-focused`, `Mui-selected`, `Mui-checked`, `Mui-expanded`, `Mui-hovered`, `Mui-pressed`, `Mui-open`, and `Mui-error`; `:hovered`, `:focus`, `:active`, `:disabled`, `:checked`, `:selected`, `:open`, `:dragging`, and `:error` style selectors now resolve from node state props.

M1 slot bridge: named child mounts now retain their MUI slot identity in `slot_attributes.mui_slot`. `mui_slot_props.root` is merged into the owning node before class/state generation, and each named slot child receives `Mui<Owner>-<slot>` plus the matching `mui_slot_props.<slot>` map before its own stylesheet matching. `mui_slots.<slot>` is recorded as `mui_slot_component` on the slot child; this preserves the replacement intent for future component-specific renderers without pretending that the current template compiler can hot-swap an already expanded node type.

M1 SVG icon contract: the retained host visual asset resolver now recognizes local `dev/material-ui/packages/mui-icons-material/lib/*.js` modules through aliases such as `mui:Add` and `@mui/icons-material/Add`. The parsing logic lives in `visual_assets/mui_icons.rs`, extracts path elements into a synthetic 24x24 SVG, then reuses the existing raster-size, tint, and cache path; no icon files are vendored into `zircon_editor/assets`, and no Slint or React dependency is introduced.

M2 primitive renderer bridge: retained template nodes now carry `component_variant`, numeric `value_number`/`value_percent`, and `open`/`popup_open` state from both v2 view projection and runtime pane projection. The native painter uses those fields for MUI Web feedback primitives without adding Slint dependencies: `Progress`/`ProgressBar`/`LinearProgress` draw deterministic or indeterminate linear tracks, `CircularProgress`/`Spinner` draw a retained circular ring, `Skeleton` supports rounded/text/circular plus static wave highlight, and `Backdrop` follows the MUI Web open/invisible contract with a 50% black scrim. Runtime layout and render-dirty classification now recognize the same MUI feedback primitive names.

M2 overlay surface defaults: retained projection now recognizes `Paper`, `Modal`, `Dialog`, `AlertDialog`, `Popover`, `Popper`, `Tooltip`, `Snackbar`, `Menu`, and `Drawer` as MUI overlay/surface roles. M2 modal dismissal contract: Dialog, Modal, Popover, and Menu descriptors now expose MUI-like focus and dismissal metadata: `disable_auto_focus`, `disable_enforce_focus`, `disable_restore_focus`, `disable_escape_key_down`, `close_on_backdrop_click`, `keep_mounted`, and ARIA label references. This keeps the MUI renderer/input gate extensible before portal stacking, transition, and focus-trap logic are added.
