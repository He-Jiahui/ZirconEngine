# Material UI Token And Component Audit

## Purpose

本文件记录 M2.1a/M2.1b/M2.2b：先把 `dev/slint` Material 参考与当前 Zircon `.ui.toml` Material token、style class、meta component、runtime layout 支撑放在同一张表里，再把 token role convergence 和 meta component root forwarding 落到 `editor_material.ui.toml`、`material_meta_components.ui.toml`、shared `.ui.toml` compiler token resolver 与 focused tests。

2026-05-23 起，本文件同时记录 ZirconEditor 的 MUI Web parity token 基线。`editor_material.v2.ui.toml` 保留既有 `material_*` 与 `slint_material_*` token，但新增 `mui_web_*` token 分组，从本地 `dev/material-ui` v9.0.1 的 palette、typography、shape、spacing、shadows、z-index 和 transitions 源文件冻结默认值，供后续组件实现选择 strict MUI Web 视觉基线。编辑器 visual asset resolver 现在可通过 `mui:Add`、`@mui/icons-material/Add` 等别名读取本地 `dev/material-ui/packages/mui-icons-material/lib/*.js`；具体解析留在 `visual_assets/mui_icons.rs`，把 MUI path module 合成为 24x24 SVG 后复用现有 raster/tint/cache 管线。运行时 Material foundation 描述器同时新增 common `mui_*` customization surface：`mui_variant`、`mui_color`、`mui_size`、`mui_slots`、`mui_slot_props`、`mui_sx`、`mui_classes`，用于承接 MUI Web 的 variant/color/size、slots、slotProps、sx 与 classes 自定义能力。运行时 style bridge 现在会把 `mui_sx` 合并为高优先级样式覆盖，并生成 `Mui<Component>-root`、variant/color/size utility class、`mui_classes` 和 MUI state class，使 `.Mui-*` 与 `:state` selector 能在模板编译阶段命中。slot bridge 同步保留命名 mount 的 `mui_slot` 身份，把 `mui_slot_props.root` 应用到 owner，把 `mui_slot_props.<slot>` 和 `Mui<Owner>-<slot>` utility class 应用到命名 slot 子节点，并用 `mui_slot_component` 记录 `mui_slots.<slot>` 的替换意图。

M2 native customization alias slice keeps the internal `mui_*` bridge but also exposes the MUI Web prop names directly: `sx`, `slotProps`, `slots`, `classes`, and `className`. The style compiler now merges native aliases with `mui_sx`/`mui_slot_props`/`mui_slots`/`mui_classes`, applies root and named-slot props before class/state selector matching, and adds `className`, `classes.root`, and `classes.<slot>` to the selector-visible class list. Runtime mutation dirty classification covers both spellings, so the future editor authoring surface can present MUI vocabulary while the retained runtime still owns the data model and keeps the no-extra-Slint-library boundary.

M2 feedback metadata slice closes the next small Web parity gap for Alert and Snackbar. Alert now carries `severity` options, `variant = standard|filled|outlined`, optional `color`, `icon`, `show_icon`, `iconMapping`, and `closeText`; Snackbar carries `message`, `autoHideDuration`, `resumeHideDuration`, snake_case timer aliases, `disableWindowBlurListener`, and `anchorOrigin` plus snake_case anchor fields. M2 feedback visual defaults slice moves Alert/Snackbar from metadata-only toward retained visual parity. 当前审计保持四个边界：

- Slint Material 是编辑器视觉与组件状态参考，不复制 Slint API。
- Zircon 的行为真源仍是 `.ui.toml` meta component + shared `UiSurfaceFrame` + runtime layout/render/input DTO。
- M2.1b/M2.2b 修改 shared token resolver、Material `.ui.toml` token/style assets、focused tests 和文档；不修改 sibling-owned native host 或 runtime layout implementation。
- M2 MUI primitive bridge 可以触达 retained host DTO、view projection、native painter 以及最小 runtime layout/mutation 支撑，但仍不改变 Cargo 依赖或引入 Slint 组件库。

## Runtime Support Boundary

`zircon_runtime/src/ui/layout/pass/material.rs` currently treats a node as Material-layout-aware when it has authored layout attributes and a supported component role. The supported role set already includes:

- controls: `Button`, `IconButton`, `ToggleButton`, `Checkbox`, `InputField`, `TextField`, `ComboBox`, `RangeField`, `NumberField`, `Switch`
- lists and menus: `ListRow`, `MenuItem`, `ContextActionMenu`, `TableRow`, `VirtualList`, `Tab`
- status and generic roles: `ProgressBar`, `Spinner`, `Label`
- editor value roles: `ColorField`, `Vector2Field`, `Vector3Field`, `Vector4Field`

This is enough for M2.1 token unification and M2.2 focused behavior tests without adding another layout path.
