核查优先级

  1. Shell 骨架稳定Topbar、Sidebar、Workspace、Bottom strip 的比例先固定。
  2. Workspace 内容换页导航只替换主内容区，不改变外壳骨架。
  3. 弹层不挤压布局Source Engine、User Menu、Filter、Sort、Delete Confirm 都是覆盖层。
  4. 状态页复用容器Empty、Loading、Error 使用同一工作区状态画布。

结构响应策略

Desktop 1568x1003

完整 Topbar + 223px Sidebar + Workspace + Bottom strip。

Compact Window

工作区内容优先压缩，外壳尺寸和层级不重新定义。

Collapsed Sidebar

侧栏只缩窄为图标轨道，导航状态和底部状态仍保留。

结构不变量

Topbar 只负责全局 source、任务状态、账号和窗口控制。 Sidebar 只负责主导航和 Engine Status 状态卡。 Workspace 负责页面标题、工具条、卡片、表格和页面内容。 Bottom strip 只负责控件状态视觉样例。 Overlays 浮在 shell 上方，不占用布局空间。

总体结构验收顺序

1. 框架比例 2. 导航归属 3. 工作区换页 4. 弹层层级 5. 状态页复用 6. 局部内容密度
