# Zircon Engine Style SVG Resources

`zircon_engine_style` is a Zircon-owned SVG icon family for engine editor concepts that go beyond shell chrome. It is informed by the categories present in the reference engine trees, but it does not copy or convert their art.

## Reference Routing

Fyrox is the primary reference for this pass because it has a Rust-native editor/runtime split and a large `editor/resources` icon vocabulary for scene objects, asset types, viewport tools, animation tracks, UI widgets, and common editing operations. Godot is the secondary reference for scalable SVG conventions, especially compact UI controls, file/resource browser states, grid/snap icons, theme icons, and script/resource affordances. Unreal Engine is the scale reference for long-lived engine families such as plugins, modules, packaging, deployment, profiling, behavior graphs, and blueprint-style authoring.

## Style Rules

The generated icons use a dark-editor friendly line language: neutral strokes use `#cbd2dc`, active Zircon accents use `#26d8d1`, warning/lighting accents use `#f0b545`, and destructive or stop/error states use `#e16666` or `#ff705f`. Most icons use a `24x24` view box with 1.3-1.7 px strokes so they can sit beside the existing Editor shell icons without visual weight jumps.

The pack is grouped by engine concept rather than by a source reference engine:

- `assets/` covers resource browser and asset-import concepts: mesh, material, texture, shader, animation clip, audio clip, scene file, prefab, script file, font, tilemap, and particle system.
- `scene/` covers hierarchy and component concepts: entity, component, camera, directional/point/spot lights, terrain, navmesh, collider, rigid body, trigger volume, skeleton, and bone.
- `graph/` covers visual scripting and graph authoring: blueprint, event node, function node, variable, state machine, transition, behavior tree, blackboard, and shader graph.
- `tools/` covers editor tool states: pipette, paint, measure, focus selection, isolate, pivot, align, and snap grid.
- `runtime/` covers preview and diagnostics controls: play in editor, simulate, pause, stop, profiler, and diagnostics.
- `build/` covers production pipeline concepts: plugin, module, package, deploy, source control, test, and cloud build.

All files are self-contained SVG. They do not reference raster images, external sprite sheets, remote URLs, or imported symbols.
