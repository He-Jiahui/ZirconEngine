# Shared Slate UI Surface Frame

## Scope
- Milestones 1-4 of `.codex/plans/Shared Slate-Style UI Layout, Render, And Hit Framework.md` establish shared visibility policy, arranged-tree render/hit coherence, hit-grid diagnostics, pointer route state, and editor-native toolbar/pane hit-grid consumption.
- Affected layers are `zircon_runtime_interface::ui` contracts, `zircon_runtime::ui` retained tree / arranged tree / render extract / hit grid behavior, and the editor native shell boundary for drawer/header/toolbar/pane hit semantics.

## Baseline
- The active worktree already contains a partial Slate-style arranged/render/hit implementation plus unrelated active editor native text/input changes.
- The known lower-layer risk before this slice was mixed visibility authority: some paths could still read `state_flags.visible` directly while others consumed `UiVisibility`, letting render, focus, scroll, and hit-test disagree.

## Test Inventory
- Interface contract: `ui_visibility_contract_separates_layout_render_and_hit_policy` covers `Visible`, `Hidden`, `Collapsed`, `HitTestInvisible`, `SelfHitTestInvisible`, and legacy bool normalization.
- Runtime coherence: `surface_rebuild_derives_render_and_hit_from_same_arranged_geometry` covers render/hit frame coherence from one arranged tree.
- Runtime hit policy: `hit_grid_respects_slate_visibility_and_clip_semantics` covers overlap, hidden/collapsed filtering, subtree blocking, and self-hit-test-invisible child hits.
- Runtime legacy boundary: `legacy_visible_false_is_normalized_into_hidden_visibility_for_surface_outputs` covers `state_flags.visible=false` normalization before arranged/render/hit output.
- Runtime layout visibility boundary: `explicit_collapsed_visibility_preserves_layout_collapse_with_legacy_visible_false` covers explicit `Collapsed` preserving layout-collapse semantics even when legacy `visible=false` is present.
- Runtime support behavior: `focus_navigation_and_scroll_candidates_use_effective_visibility` covers focus/navigation and scroll candidates using the same effective visibility helpers.
- Runtime hit-grid diagnostics: `hit_grid_omits_disabled_nodes_and_debug_dump_reports_why`, `scrollable_virtualized_children_enter_hit_grid_only_when_arranged_visible`, `surface_frame_carries_focus_capture_and_hover_snapshot`, and `surface_dirty_rebuild_keeps_render_only_changes_out_of_layout` cover disabled filtering, shared `UiSurfaceFrame` hit/debug helpers, virtualized scroll visibility, focus/capture/hover snapshots, and dirty rebuild separation.
- Runtime pointer routing: `pointer_routing_tracks_hover_focus_capture_and_activation_phase` covers `UiPointerRoute.hit_path` preserving the shared root-to-leaf and leaf-to-root hit route used by dispatch.
- Editor toolbar projection: `viewport_toolbar_surface_frame_includes_projected_route_controls_without_action_list` covers deriving toolbar hit nodes from projected `.ui.toml` route-bearing control frames instead of maintaining a Rust action-coordinate list.
- Editor toolbar native cutover: `native_host_pointer_click_routes_viewport_toolbar_buttons_before_viewport_body`, `native_host_viewport_toolbar_only_dispatches_primary_press`, and `native_host_pointer_click_routes_late_viewport_toolbar_controls` cover the native host consuming `UiSurfaceFrame.hit_grid` frames before viewport body routing and preserving primary-press-only activation.
- Editor projection fallback: `shared_viewport_toolbar_pointer_click_falls_back_to_surface_projection_when_control_rect_is_empty` covers TOML projection ids dispatching through template bindings with no legacy toolbar route alias.
- Editor pane projection: `native_host_pointer_click_routes_pane_template_button_actions` and `native_host_pointer_click_routes_binding_only_template_buttons` cover native pane template clicks going through submitted `PaneData.body_surface_frame` hit grids instead of rebuilding local hit coordinates during dispatch.
- Editor native fast path: `presenter::tests` covers damage-only softbuffer copy and damage clamping, `host_contract::redraw::tests` covers request coalescing semantics, `builtin_template_compile_cache_is_reused_across_runtime_instances` covers process-local builtin template reuse, and the native host contract module covers viewport move idle routing plus hierarchy/asset hover fast paths.
- Editor regression boundary: visible drawer projection and toolbar hit tests must be rerun after this slice because the editor should consume shared visibility and arranged geometry instead of local bool-visible policy.
